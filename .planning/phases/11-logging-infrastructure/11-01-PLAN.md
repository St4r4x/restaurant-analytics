Step 1 — Create src/main/resources/logback-spring.xml with the following exact content:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<configuration>

  <!-- ================================================================ -->
  <!-- Non-prod profile: human-readable plaintext with [requestId] in   -->
  <!-- every log line. MDC key "requestId" is populated by              -->
  <!-- RequestIdFilter (Phase 11 Plan 02). Empty string when no         -->
  <!-- request context (e.g., application startup logs).               -->
  <!-- ================================================================ -->
  <springProfile name="!prod">
    <appender name="CONSOLE" class="ch.qos.logback.core.ConsoleAppender">
      <encoder>
        <pattern>%d{HH:mm:ss.SSS} %-5level %logger{36} [%X{requestId}] - %msg%n</pattern>
      </encoder>
    </appender>

    <!-- Third-party noise reduction: suppress MongoDB driver and BSON verbosity -->
    <logger name="org.mongodb.driver" level="WARN"/>
    <logger name="org.bson" level="WARN"/>
    <logger name="org.springframework.web" level="INFO"/>
    <logger name="org.springframework.security" level="INFO"/>

    <!-- Application code: DEBUG in dev for full visibility -->
    <logger name="com.aflokkat" level="DEBUG"/>

    <root level="INFO">
      <appender-ref ref="CONSOLE"/>
    </root>
  </springProfile>

  <!-- ================================================================ -->
  <!-- prod profile: Logstash-compatible JSON (one object per line).    -->
  <!-- LogstashEncoder automatically includes all MDC fields (including  -->
  <!-- "requestId") as top-level JSON keys — no special pattern needed. -->
  <!--                                                                   -->
  <!-- logstash-logback-encoder 7.3 — PINNED. Do NOT upgrade:           -->
  <!-- 7.4+ requires Logback 1.3.x; Spring Boot 2.6.15 ships 1.2.12.   -->
  <!-- ================================================================ -->
  <springProfile name="prod">
    <appender name="JSON_CONSOLE" class="ch.qos.logback.core.ConsoleAppender">
      <encoder class="net.logstash.logback.encoder.LogstashEncoder"/>
    </appender>

    <!-- Same third-party noise reduction as dev -->
    <logger name="org.mongodb.driver" level="WARN"/>
    <logger name="org.bson" level="WARN"/>
    <logger name="org.springframework.web" level="INFO"/>
    <logger name="org.springframework.security" level="INFO"/>

    <!-- Application code: INFO in prod to reduce log volume -->
    <logger name="com.aflokkat" level="INFO"/>

    <root level="INFO">
      <appender-ref ref="JSON_CONSOLE"/>
    </root>
  </springProfile>

</configuration>
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| log output → log aggregator | Log lines leave the JVM process; any sensitive data in MDC fields would appear in log aggregators or stdout captures |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Information Disclosure | logback-spring.xml / LogstashEncoder | mitigate | LogstashEncoder serializes ALL MDC fields as top-level JSON keys. Mitigation: only put non-sensitive identifiers in MDC (requestId is a UUID with no semantic content). Never put JWT tokens, passwords, or PII in MDC. Document this constraint in logback-spring.xml comments. |
| T-11-02 | Tampering | logback-spring.xml / springProfile tag | accept | If file is renamed to logback.xml (wrong name), springProfile tags are silently ignored and both appenders may activate. Risk: logging misconfiguration only — no security boundary crossed. Mitigation note in file comments. |
| T-11-03 | Information Disclosure | prod log output | mitigate | JSON logs may expose internal class names, stack traces, and error messages to operators with log access. Mitigation: set MongoDB driver and BSON at WARN in prod (reduces internal query structure leakage). Spring Security at INFO (no credential data logged at INFO level). |
</threat_model>
