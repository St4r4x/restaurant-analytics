<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| .env file → docker compose → container env | JWT_SECRET, POSTGRES_PASSWORD travel from developer's .env into container environment at `docker compose up` time |
| docker-compose.yml → git repository | This file is tracked; any hardcoded value becomes part of git history permanently |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-09 | Information Disclosure | `docker-compose.yml` — `CONTROLLER_SIGNUP_CODE: changeme` hardcoded | mitigate | Replace with `${CONTROLLER_SIGNUP_CODE:-}` — value sourced from .env which is gitignored |
| T-13-10 | Information Disclosure | `docker-compose.yml` — `POSTGRES_PASSWORD: restaurant` hardcoded | mitigate | Replace with `${POSTGRES_PASSWORD}` — required variable, startup fails descriptively if .env is missing |
| T-13-11 | Denial of Service | Container with no memory limit can consume all host memory, crashing sibling containers | mitigate | `deploy.resources.limits.memory` set on all 4 services: 512m/512m/128m/256m |
| T-13-12 | Elevation of Privilege | Docker Compose container-to-container network without JWT enables bypass of auth if JWT_SECRET is empty | mitigate | `JWT_SECRET: ${JWT_SECRET}` (required, no default) ensures AppConfig assertion fires at startup if .env is misconfigured |
</threat_model>
