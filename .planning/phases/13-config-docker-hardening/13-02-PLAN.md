<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| container user → host | App container running as root can escape to host if container escape is exploited |
| build context → Docker daemon | Files sent to Docker daemon during build; sensitive files in context could be leaked via image layers |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-06 | Elevation of Privilege | App container running as root (no USER directive) | mitigate | Add `RUN addgroup -S appuser && adduser -S appuser -G appuser` and `USER appuser` before ENTRYPOINT in Dockerfile runtime stage |
| T-13-07 | Information Disclosure | Docker build context includes `.env` (secrets), `.git/` (history), `.planning/` (internal notes) | mitigate | Create `.dockerignore` explicitly excluding `.env`, `.env.*`, `.git/`, `.planning/` — these must never be baked into image layers |
| T-13-08 | Information Disclosure | Old Java 21 runtime (`eclipse-temurin:21-jre-jammy`) would cause UnsupportedClassVersionError — attacker sees stack trace with version info | mitigate | Upgrade to `eclipse-temurin:25-jre-alpine` matching pom.xml compiler target — also reduces attack surface vs jammy's larger package set |
</threat_model>
