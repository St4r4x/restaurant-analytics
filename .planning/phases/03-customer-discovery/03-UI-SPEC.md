## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (0.88em base) | 400 regular | 1.5 | Table cells, list item meta, API descriptions |
| Label | 13px (0.85em) | 500 medium | 1.4 | Cuisine tags, stat labels, muted meta text |
| Heading (card) | 20px (1.25em) | 600 semibold | 1.2 | Card `h2` titles (color: #667eea) |
| Display | 29px (1.8em) | 700 bold | 1.1 | Restaurant name on detail page (`.restaurant-name`) |

Rules:
- Exactly 2 weights in use: 400 (regular) and 600–700 (semibold/bold). Do not introduce 300 or 500 as new weight values in Phase 3 HTML.
- All font sizes expressed as `em` relative to the `body` base, matching the existing templates. Do not switch to `rem` or `px` literals for font sizes in new code.
- Grade badges and score values use `font-weight: 700` always.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#667eea` → `#764ba2` gradient | Page background on all screens |
| Secondary (30%) | `#ffffff` (white cards), `#f8f9ff` (row backgrounds) | Cards, list rows, table rows |
| Accent (10%) | `#667eea` solid | See reserved-for list below |
| Destructive | `#c33` / `#ef4444` | Error states, grade C/F markers only |

**Accent (#667eea) reserved for:**
- Card heading color (`card h2`)
- Card bottom border underline (`border-bottom: 3px solid #667eea`)
- Primary action buttons (`background: #667eea`)
- Input focus ring (`border-color: #667eea`)
- Search input active state border
- Leaflet map user-position marker fill
- Spinner top border (`border-top: 3px solid #667eea`)
- Bookmark icon when in saved state (use `#f5a623` amber, NOT accent — this is already established in index.html `.btn-bookmark.bookmarked`)

**Grade color contract (consistent across ALL Phase 3 surfaces — map markers, detail badge, search result card badges, bookmarks list):**

| Grade | Background | Text / Border | Hex literal to use |
|-------|------------|---------------|--------------------|
| A | `#e8f5e9` | `#2e7d32` / `#4caf50` | marker fill: `#22c55e` |
| B | `#fff8e1` | `#f57f17` / `#ffc107` | marker fill: `#eab308` |
| C or F (and any other) | `#ffebee` | `#b71c1c` / `#f44336` | marker fill: `#ef4444` |

Note: Leaflet map markers (circle divIcon) use the flat hex `#22c55e` / `#eab308` / `#ef4444` as confirmed by CONTEXT.md and project instructions. The badge pill CSS on `restaurant.html` uses the softer tint/border variant. Both must remain consistent — same grade maps to same color family.

## Component Inventory for Phase 3

These are the specific UI components added or enhanced in this phase. Each is described as a design contract.

### 1. Search bar (added to `index.html`)

- **Placement:** At the top of the `.container`, between `<header>` and the first `.dashboard` grid row
- **Input appearance:** Full-width within a `.card`; input matches existing `input` style (`padding: 7px 11px; border: 1px solid #ddd; border-radius: 6px; font-size: 0.88em; background: #fafafa`); focus: `border-color: #667eea` with `box-shadow: 0 0 5px rgba(102,126,234,0.2)`
- **Placeholder text:** `Search by restaurant name or address…`
- **Debounce:** 300ms `setTimeout` / `clearTimeout` pattern inline in `<script>`
- **Loading state:** Replace results list with `.spinner` div while fetch is in flight
- **Results list:** Rendered below the input inside the same card; max 20 items; each item is a `.top-restaurant-item` row (reuse existing class)
- **Empty query:** Results container hidden (`display: none`), normal dashboard content visible
- **No results:** Show `<p class="error">No restaurants found for "{query}". Try a different name or address.</p>`

### 2. Search result card row

Reuses `.top-restaurant-item` pattern from index.html:

```
[restaurant name]           [grade badge]  [Bookmark ★]  [View →]
[borough · cuisine]
```

- Grade badge: inline pill using the grade color contract above
  - `display: inline-block; padding: 2px 8px; border-radius: 12px; font-weight: 700; font-size: 0.82em`
  - Grade A: `background: #e8f5e9; color: #2e7d32`
  - Grade B: `background: #fff8e1; color: #f57f17`
  - Grade C/F/other: `background: #ffebee; color: #b71c1c`
- Bookmark toggle: `.btn-bookmark` star button, reuse existing class (amber `#f5a623` when saved, `#ccc` when not)
- "View" link: `.btn-view` anchor navigating to `/restaurant/{id}`, reuse existing class

### 3. Grade badge circle (on `restaurant.html` detail page)

- Large format: `width: 56px; height: 56px; border-radius: 50%; display: flex; align-items: center; justify-content: center; font-size: 1.8em; font-weight: 700`
- Placed in the right portion of `.header-card`, flex-shrunk alongside `.header-info`
- Color mapping: same grade color contract as above
- Shows the letter grade only (A / B / C / F); score is displayed separately as a numeric value beneath it

### 4. Leaflet grade-colored markers (on `inspection-map.html`)

- Use `L.divIcon` with inline `<div>` styled as a circle
- Circle size: `width: 12px; height: 12px; border-radius: 50%`
- Grade A fill: `#22c55e`; Grade B fill: `#eab308`; Grade C/F fill: `#ef4444`
- White border: `border: 2px solid #fff`; drop shadow: `box-shadow: 0 1px 4px rgba(0,0,0,0.4)`
- Cluster plugin: `Leaflet.markerCluster` via CDN (see Registry section)
- Cluster bubble style: default markerCluster CSS; cluster count text color `white`; cluster backgrounds use accent `#667eea` for small clusters, `#764ba2` for medium, `#ef4444` for large (default markerCluster class overrides in `<style>`)

### 5. Map marker popup

Content layout:
```
[Restaurant name — bold]
Grade: [grade badge pill]
[View details →]  (link to /restaurant/{id})
```

- Max popup width: `200px`
- Font size: `0.88em`
- Link style: `color: #667eea; font-weight: 600`

### 6. Bookmark toggle button (on `restaurant.html` detail page)

- Position: inside `.header-card`, right side, below the grade badge circle
- Unsaved state: `+ Bookmark` — button styled as `.nav-link`-variant (ghost pill: `background: rgba(255,255,255,0.15)` on purple bg OR `background: #f0f2ff; color: #667eea; border: 1px solid #c5caf7` on white card surface)
- Saved state: `✓ Saved` — filled: `background: #f5a623; color: white; border-color: #f5a623`
- Unauthenticated click: `window.location.href = '/login'` immediately, no API call

### 7. `/my-bookmarks` page

- Extends the overall page shell: same `body` gradient, `.container`, `<header>` pattern
- Back link: `← Dashboard` (same `.back-link` style as `restaurant.html`)
- Grid: single `.dashboard-wide` column card containing the bookmark list
- List items: `.top-restaurant-item` rows — name, borough, cuisine, grade badge pill, "View" link, remove-bookmark star button
- Empty state: inside the card, `<p style="color:#aaa;font-size:0.9em;padding:20px 0;text-align:center">You have no saved restaurants yet. <a href="/" style="color:#667eea;font-weight:600">Search for restaurants</a> to bookmark them.</p>`

## Interaction Contract

### Search debounce
- Minimum query length before firing: 2 characters
- Debounce: 300ms (`clearTimeout` / `setTimeout` pattern)
- On query clear (length < 2): cancel any pending fetch, hide results div

### Bookmark toggle
- Optimistic UI: toggle button state immediately on click; revert on API error
- JWT absent: redirect to `/login` before any fetch
- POST/DELETE: `fetchWithAuth('/api/users/me/bookmarks/{restaurantId}', { method })` — reuse `fetchWithAuth` pattern from index.html
- On 401 response: `fetchWithAuth` already redirects to `/login` via `handleFetchErrorResponse`

### Map clustering
- Load once on page init: `fetch('/api/restaurants/map-points')` — no auth required (public endpoint per Phase 1 decisions)
- Render all markers into a `L.markerClusterGroup()` layer, add to map
- No re-fetch on zoom — clustering is client-side only

### Loading states
- Any async fetch: show `.spinner` in the target container before fetch, replace on success or error
- Error state: `.error` div with problem description + action hint
