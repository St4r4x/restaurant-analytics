<interfaces>
<!-- Navbar insertion pattern (from Plan 02 — fragments/navbar.html). -->

Insertion rule per template:
1. Add `<div th:replace="fragments/navbar :: navbar"></div>` as the FIRST child of `<body>`
2. Find the `.container` div (has class="container") and add `style="padding-top: 72px"` to it
   - If the container already has a `style` attribute, APPEND `padding-top: 72px` to the existing value
   - If no style attribute exists, add `style="padding-top: 72px"`

These are the ONLY two changes per template. Nothing else changes: no JS, no existing CSS, no card markup, no heading text, no API endpoints.

The UI-SPEC page structure pattern:
```html
<body>
  <div th:replace="fragments/navbar :: navbar"></div>  <!-- INSERTED LINE 1 -->
  <div class="container" style="padding-top: 72px">    <!-- MODIFIED: added padding-top -->
    <header>...</header>
    <!-- existing content untouched -->
  </div>
</body>
```
