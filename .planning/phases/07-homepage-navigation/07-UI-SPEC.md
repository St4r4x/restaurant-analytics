## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, grade badge internal padding, inline list separators |
| sm | 8px | Button padding vertical, gap between sibling badges/tags |
| md | 16px | Default card internal element spacing, form field margin-bottom |
| lg | 24px | Card padding (horizontal), section heading padding-bottom |
| xl | 32px | Inter-section gap within a page |
| 2xl | 48px | Hero top/bottom padding |
| 3xl | 64px | Not used in this phase |

**Source:** Measured from existing templates. `dashboard.html` uses padding:20px 24px on .card; analytics.html uses padding:24px 16px on .container.

Exceptions:
- Navbar height: 56px total (16px padding top + bottom + content) — fixed bar, sits above `.container`
- Nav pill buttons: `padding: 8px 16px` (uses `sm=8px` vertical, `md=16px` horizontal from scale)
- Touch targets on all interactive nav elements: minimum 44px height on mobile

## Component Inventory

### New components in this phase

#### 1. Navbar fragment (`fragments/navbar.html`)

```
[ Logo text ] ··· [ Search ] [ Map ] [ Analytics ] ··· [ Sign In ] or [ {username} | Sign Out ]
```

- Height: 56px fixed
- Background: `linear-gradient(135deg, #667eea 0%, #764ba2 100%)`
- Logo: text "NYC Inspections" left-aligned, font-size 1.1em, weight 700, color white
- Nav links (center): pill style — `padding: 8px 16px; background: rgba(255,255,255,0.15); border-radius: 6px; color: rgba(255,255,255,0.85); font-size: 0.82em`
- Active page: pill background `rgba(255,255,255,0.28)`, border `1px solid rgba(255,255,255,0.5)`
- Auth area (right): placeholder `<span id="nav-auth">` filled by JS on DOMContentLoaded
  - Logged out: `<a href="/login" class="btn btn-primary btn-sm">Sign In</a>`
  - Logged in: `<a href="/profile" style="color:rgba(255,255,255,0.9);font-size:0.85em">{username}</a>` + `<button onclick="logout()" class="btn btn-secondary btn-sm">Sign Out</button>`
- Username decoded from JWT: `JSON.parse(atob(localStorage.getItem("accessToken").split('.')[1])).sub`
- Navbar sits above the `.container` div (not inside it). Top-level `<nav>` element.
- Thymeleaf insertion: `th:replace="fragments/navbar :: navbar"` inserted as first child of `<body>` in every template except `login.html`

#### 2. Landing page (`landing.html`) — three sections

**Section 1: Hero**
- Full-width, gradient background (body already has gradient — hero uses a centered text block)
- City name: "NYC Restaurant Inspector"
- Tagline: "Search 27,000+ restaurants. Know before you go."
- Stat strip: 2 live values from `/api/analytics/kpi` — "X restaurants tracked · Y% Grade A"
- Loading state: `—` em dash while fetch in flight (no change to existing pattern)
- Centered, color white, padding 48px 16px

**Section 2: Search CTA**
- White `.card` below the hero
- Card heading: "Search Restaurants"
- Input: full-width text input, `placeholder="Restaurant name or address…"`, debounce 300ms
- Button label: "Search Restaurants"
- Results: inline list of result rows below input — each row: `name · borough · grade badge · link to /restaurant/{id}`
- Empty results: "No restaurants found for '{query}'. Try a different name or address."
- Error (fetch failed): "Could not load results. Check your connection and try again."

**Section 3: Sample restaurants strip**
- White `.card` below search card
- Card heading: "Discover Restaurants"
- 3 cards in a `display: grid; grid-template-columns: repeat(3, 1fr); gap: 16px`
- Each card: name (body weight 400), borough (label gray `#666`), grade badge, `<a href="/restaurant/{id}">View Details</a>` link
- Data from `GET /api/restaurants/sample?limit=3`
- Loading: 3 placeholder rectangles with `background: #f0f2ff; border-radius: 8px; height: 80px` (no animated skeleton — that is Phase 9)
- Empty/error: "Could not load sample restaurants. Refresh to try again."

#### 3. Personalised customer homepage (`index.html` rewrite)

Three strips stacked vertically inside `.container`:

**Strip A: Recent Bookmarks**
- Section heading `h3`: "Your Bookmarks"
- 3 bookmark cards horizontal (same grid as sample restaurants: 3 columns, gap 16px)
- Each card: name, borough, grade badge, link to `/restaurant/{id}`
- Empty state: "You haven't bookmarked any restaurants yet. Find one to save →" with link to `/`
- Data from existing bookmarks endpoint

**Strip B: Nearby Restaurants**
- Section heading `h3`: "Nearby"
- Only rendered if `navigator.geolocation` is available and permission granted
- If denied or unavailable: section hidden entirely (no empty state message — section simply does not render)
- Up to 5 nearby restaurant cards (same card pattern)
- Uses existing Leaflet-based geolocation from current `index.html` nearby-map section — migrated here as card list, not a map

**Strip C: City KPI Tiles**
- 4 tiles in `display: grid; grid-template-columns: repeat(4, 1fr); gap: 16px`
- Identical to analytics.html KPI tiles (reuse exact markup pattern)
- Data from `GET /api/analytics/kpi`

#### 4. Profile page (`profile.html`)

- Centered white `.card` (max-width 480px, margin: 48px auto)
- Avatar: Unicode person icon `&#128100;` (or similar), 48px, color `#667eea`, centered
- Username: Display size (1.5em), weight 700, color `#333`, centered
- Email: Body size, weight 400, color `#666`, centered
- Role badge: pill — CUSTOMER green `#22c55e`, CONTROLLER orange `#f97316`, white text, `border-radius: 20px; padding: 4px 12px; font-size: 0.78em; font-weight: 700`
- Stats row: `Bookmarks: {N}` · `Reports: {N}` (reports only shown when role is CONTROLLER)
- Stats displayed as label (0.82em, weight 400, color #555) + value (body, weight 700, color #333)
- Data from `GET /api/users/me`
- Loading: `—` em dash while fetch in flight

## Interaction Contracts

### Navbar JS behavior

| Trigger | Behavior |
|---------|----------|
| Page load | Read `localStorage.getItem("accessToken")`. If token present and not expired, decode `.sub`, render logged-in state. If absent or expired, render logged-out state. |
| Sign Out button click | Call `localStorage.removeItem("accessToken")` + `localStorage.removeItem("refreshToken")`, redirect to `/login` |
| `/profile` link click | Navigates to `/profile`; if no token in localStorage, redirect to `/login` instead |
| Active link detection | Compare `window.location.pathname` to each nav link href; apply active pill style to matching link |

### Landing page search

| Trigger | Behavior |
|---------|----------|
| Input keystroke | Debounce 300ms, then `GET /api/restaurants/search?q={value}`. Clear results if input empty. |
| Result row click | Navigate to `/restaurant/{id}` |
| Fetch error | Show error copy inline below input, hide result list |
| Zero results | Show "No restaurants found…" copy, no result list |

### Sample restaurants (landing.html)

| Trigger | Behavior |
|---------|----------|
| Page load | `GET /api/restaurants/sample?limit=3`. Show loading placeholder while in flight. |
| Fetch error | Show "Could not load sample restaurants. Refresh to try again." inline in the section |

### Profile page

| Trigger | Behavior |
|---------|----------|
| Page load | Read token from localStorage. If absent, `window.location.href = '/login'`. If present, `GET /api/users/me` with Bearer header. |
| `reportCount` is null (CUSTOMER) | Hide reports stat row entirely |
| Fetch error | Show "Could not load profile. Please log in again." with a "Sign In" link |

## Page Structure Pattern

All pages (existing and new) follow this exact structure:

```html
<body>
  <nav>  <!-- navbar fragment, 56px fixed height -->
  <div class="container" style="padding-top: 72px">  <!-- 56px nav + 16px gap -->
    <header>  <!-- page-specific, white text on gradient -->
    <!-- section cards -->
  </div>
</body>
```

The `padding-top: 72px` on `.container` is the only structural change to existing templates when navbar is inserted. All existing card/grid/section patterns are preserved as-is.
