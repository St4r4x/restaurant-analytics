<task type="auto">
  <name>Task 2: Clean application.properties of removed Boot 3+ properties</name>
  <files>src/main/resources/application.properties</files>
  <read_first>src/main/resources/application.properties</read_first>
  <action>
Remove two property lines from application.properties:

1. Remove this entire line (including the comment above it):
   ```
   # Swagger / OpenAPI (Spring Boot 2.6 compatibility)
   spring.mvc.pathmatch.matching-strategy=ant_path_matcher
   ```
   This property is REMOVED in Boot 3+. Boot 3 uses PathPatternParser exclusively; the old AntPathMatcher strategy is not configurable via this property. springdoc 2.x does not need it.

2. Remove this line from the JPA section:
   ```
   spring.jpa.properties.hibernate.dialect=org.hibernate.dialect.PostgreSQLDialect
   ```
   Hibernate 6 (shipped with Boot 3+) auto-detects the PostgreSQL dialect. Keeping the explicit value generates a deprecation warning and is unnecessary.

Keep all other properties exactly as-is. Keep the `springdoc.swagger-ui.path=/swagger-ui.html` and `springdoc.api-docs.path=/api-docs` lines — these are valid in springdoc 2.x.

Also keep the `# Swagger / OpenAPI` section comment if it still has other content below it (springdoc.* lines), but remove the comment line itself if it only referred to the ant_path_matcher property.
  </action>

    <automated>grep -c "ant_path_matcher\|hibernate.dialect" /home/missia03/Projects/restaurant-analytics/src/main/resources/application.properties</automated>
