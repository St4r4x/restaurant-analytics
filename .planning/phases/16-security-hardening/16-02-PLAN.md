public CorsConfigurationSource corsConfigurationSource() {
    CorsConfiguration config = new CorsConfiguration();
    config.setAllowedOrigins(List.of("http://localhost:8080"));
    config.setAllowedMethods(List.of("GET", "POST", "PUT", "DELETE", "OPTIONS"));
    config.setAllowedHeaders(List.of("Authorization", "Content-Type"));

}

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser→/api/** | Cross-origin requests from browsers; CORS policy controls which origins are allowed |
| HTTP response→client | Response headers control browser behavior (framing, MIME sniffing) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-01 | Spoofing | CORS wildcard on controllers | mitigate | Remove @CrossOrigin(origins="*") from all 4 controllers; replace with CorsConfigurationSource bean whitelisting only http://localhost:8080 (per D-01/D-03) |
| T-16-02 | Tampering | Clickjacking via iframe embedding | mitigate | X-Frame-Options: DENY via http.headers().frameOptions(fo -> fo.deny()) in SecurityConfig (per D-04) |
| T-16-03 | Tampering | MIME-type sniffing enabling XSS via file uploads | mitigate | X-Content-Type-Options: nosniff via http.headers().contentTypeOptions(withDefaults()) (per D-04) |
| T-16-04 | Elevation of Privilege | Controller @CrossOrigin overriding global CORS policy | mitigate | Delete @CrossOrigin from AuthController, RestaurantController, AnalyticsController, InspectionController (per research Pitfall 1) |
</threat_model>
