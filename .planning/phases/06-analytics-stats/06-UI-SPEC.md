## Typography

All sizes expressed in em relative to base 16px browser default. Pixel equivalents provided.

**Declared scale: 4 roles, 2 weights only (400 regular, 700 bold).**

The Body/Label distinction is intentional: Body is content text (weight 400, slightly larger at 0.88em) for reading; Label is metadata/UI chrome (weight 700, slightly smaller at 0.82em) for scanning and identification. Do not conflate them.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 0.88em (~14px) | 400 regular | 1.5 | Table cell text, list items, description text |
| Label | 0.82em (~13px) | 700 bold | 1.4 | Form labels, badge text, small metadata (date, borough under card title), tab labels, nav links, table headers, KPI tile sub-label, "View ‚Üí" link |
| Heading | 1.1em‚Äì1.25em (~18‚Äì20px) | 700 bold | 1.2 | Card section headings (`.card h3`), modal title |
| Display | 2em (~32px) | 700 bold | 1.0 | KPI tile large number (e.g. "27,000", "73.4%") |

Size mapping notes:
- KPI tile label below number: maps to Label role (0.82em, weight 400 for muted metadata variant ‚Äî color `#666`)
- Navigation / subheading text (tab labels, nav links): maps to Label role (0.82em, weight 700)
- Table headers: maps to Label role (0.82em, weight 700)
- No weight 600 (semibold) anywhere in this phase; all emphasis uses weight 700

Codebase-inherited exceptions (present in existing templates but NOT reproduced in new components):
- Page header H1: 1.5em at weight 700 ‚Äî this size exists only in the header strip inherited from dashboard.html line 37; it is not a fourth scale step and must not be used for any new element in analytics.html beyond the single H1 in the header strip

Source: `dashboard.html` lines 12‚Äì37, `index.html` lines 37‚Äì55, 105, 230‚Äì238.

## Component Inventory

Components used or introduced in this phase. All implemented as inline HTML + CSS ‚Äî no external component library.

**Primary focal point:** KPI tile numbers (Display role, 2em, `#667eea`) ‚Äî the first elements visible on page load, providing the city-wide summary figures. All other components support and contextualize these numbers.

### KPI Tiles (STAT-01)

- Container: `.dashboard` grid with `grid-template-columns: repeat(4, 1fr)`, gap `16px`, auto-wraps at `<600px`
- Each tile: `.card` base style (`background: white; border-radius: 12px; padding: 20px 24px; box-shadow: 0 4px 20px rgba(0,0,0,0.12)`) plus `text-align: center`
- Number element: `<div id="kpi-total" style="font-size:2em;font-weight:700;color:#667eea">‚Äî</div>`
- Label element: `<div style="font-size:0.82em;color:#666;margin-top:4px">Total Restaurants</div>`
- Loading state: display `‚Äî` (em dash U+2014) in each number element until API data resolves
- No spinner on KPI tiles ‚Äî em dash is sufficient for loading (consistent with existing pattern)

### Borough Grade Distribution Chart (STAT-02)

- Container: `.card` wrapping a `<canvas id="boroughChart">` element
- Canvas container height: `280px` fixed (set via `style="position:relative;height:280px"` on wrapper div)
- Chart type: `bar` with `indexAxis: "y"`, `stacked: true` on both x and y axes
- Dataset colors: Grade A `#22c55e`, Grade B `#eab308`, Grade C `#ef4444`
- X-axis: percentages 0‚Äì100, tick callback appends `%`, no gridlines on y-axis
- Y-axis labels: borough names in UPPERCASE as returned by MongoDB (e.g. `MANHATTAN`)
- Legend: display at top, labels A / B / C
- Tooltip: custom callback showing `"A: 3200 (72.0%)"` format ‚Äî raw counts stored separately in JS
- Animation: `duration: 600` (default Chart.js 4 animation, no override needed)
- Loading state: canvas hidden until data arrives; replace with `‚Äî` text if fetch fails

### Cuisine Rankings (STAT-03)

- Container: `.dashboard` grid with `grid-template-columns: repeat(2, 1fr)`, gap `16px`
- Each side: `.card` with heading "Top 10 Cleanest" / "Top 10 Worst"
- List: `<ol>` styled with `padding-left: 0; list-style: none`
- Each item: flex row `rank ¬∑ cuisine name ¬∑ avg score`; rank is `font-weight: 700; color: #667eea`; score is right-aligned `font-size: 0.88em; color: #555`
- Score format: numeric rounded to 1 decimal (e.g. `12.4`) ‚Äî no color badges on scores
- Item padding: `8px 0`, `border-bottom: 1px solid #eee` on all but last

### At-Risk Table (STAT-04)

- Container: `.card` full-width
- Table: `<table style="width:100%;border-collapse:collapse">`
- Columns: Restaurant Name | Borough | Grade | View
- Header row: `font-size: 0.82em; font-weight: 700; color: #555; border-bottom: 2px solid #eee; padding: 6px 8px`
- Data rows: `font-size: 0.88em; padding: 8px; border-bottom: 1px solid #f0f0f0`; alternate row background `#fafafa` on even rows
- Grade column: `gradeBadgeHtml(grade)` function ‚Äî verbatim copy from `index.html`
- View column: `<a href="/restaurant/{id}" style="color:#667eea;font-size:0.82em;font-weight:700">View ‚Üí</a>` ‚Äî this is a table-cell navigation link, not a CTA; the row's restaurant name provides the accessible noun context; label follows the existing `dashboard.html` convention
- Fixed limit: 50 rows maximum (no pagination in this phase)
- Empty state: single row spanning all columns with "No at-risk restaurants found." in `color: #888`

### Page Header Strip

- Background: `linear-gradient(135deg, #667eea 0%, #764ba2 100%)`
- Content: H1 page title + subtitle paragraph + nav link back to home
- H1: `font-size: 1.5em; font-weight: 700; color: white` (codebase-inherited exception ‚Äî see Typography section)
- Subtitle: `font-size: 0.82em; color: rgba(255,255,255,0.8)` (Label role)
- Home link: pill style matching dashboard.html line 41 ‚Äî `background: rgba(255,255,255,0.15); border-radius: 6px; padding: 7px 14px; color: rgba(255,255,255,0.85); font-size: 0.82em`
- Nav link to Analytics in existing pages: add `<a href="/analytics">Analytics</a>` in the header of `index.html` and `dashboard.html` matching the existing nav link pill style

## Interaction Contract

### Page load sequence

1. Template renders immediately with header strip, section headings, and `‚Äî` placeholders in KPI tiles; canvas element present but empty; table shows skeleton structure with no rows.
2. On `DOMContentLoaded`: single `fetch('/api/analytics/kpis')` fires; on success populate 4 KPI divs with formatted numbers; on failure replace each `‚Äî` with error copy ("Could not load stats.").
3. On `DOMContentLoaded`: `fetch('/api/analytics/borough-grades')` fires; on success initialize Chart.js canvas; on failure hide canvas and show error copy.
4. On `DOMContentLoaded`: `fetch('/api/analytics/cuisine-rankings')` fires; on success populate both ranked lists; on failure show empty state copy in each list.
5. On `DOMContentLoaded`: `fetch('/api/analytics/at-risk')` fires; on success populate table rows; on failure show error row.
6. All 4 fetches fire concurrently (no sequential dependency).

### Number formatting

- `totalRestaurants`: integer with thousands separator ‚Äî `toLocaleString()` (e.g. `27,000`)
- `percentGradeA`: one decimal place followed by `%` (e.g. `73.4%`)
- `avgScore`: one decimal place (e.g. `14.2`)
- `atRiskCount`: integer with thousands separator ‚Äî `toLocaleString()`
- Cuisine avg score: one decimal place (e.g. `12.4`)

### Accessibility

- `<canvas>` element includes `aria-label="Borough grade distribution chart"` and `role="img"`
- Table includes `<caption>` matching the section heading text
- All links have descriptive text (not bare "click here")
- Grade badge `<span>` elements include `aria-label="Grade A"` (substitute actual grade)
