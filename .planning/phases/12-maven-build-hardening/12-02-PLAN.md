**Step 3 — Add the check execution to pom.xml:**

Inside the existing `<executions>` block of the `jacoco-maven-plugin`, add a third `<execution>` block AFTER the closing `</execution>` tag of `jacoco-report`. The complete block to insert:

```xml
        <!-- Goal 3: Fail build if INSTRUCTION coverage drops below threshold.
             Bound to 'test' phase (same as report) so plain 'mvn test' enforces it.
             Binding to 'verify' would require 'mvn verify' — would silently pass on 'mvn test'. -->
        <execution>
          <id>jacoco-check</id>
          <phase>test</phase>
          <goals><goal>check</goal></goals>
          <configuration>
            <excludes>
              <!-- Same exclusion set as jacoco-report — D-04 -->
              <exclude>com/st4r4x/dto/**</exclude>
              <exclude>com/st4r4x/entity/**</exclude>
              <exclude>com/st4r4x/aggregation/**</exclude>
              <exclude>com/st4r4x/domain/**</exclude>
            </excludes>
            <rules>
              <rule>
                <element>BUNDLE</element>
                <limits>
                  <limit>
                    <!-- Measured baseline: {baseline_pct}% — threshold set at {baseline_pct - 5}% (baseline minus 5%) -->
                    <counter>INSTRUCTION</counter>
                    <value>COVEREDRATIO</value>
                    <minimum>{minimum_ratio}</minimum>
                  </limit>
                </limits>
              </rule>
            </rules>
          </configuration>
        </execution>
```

Replace `{baseline_pct}`, `{baseline_pct - 5}`, and `{minimum_ratio}` with the actual values from Step 1 and 2. Do NOT use template placeholders in the committed file.

**Step 4 — Verify the check goal fires correctly:**

Run `mvn test` — must exit 0 (coverage is at baseline, threshold is baseline minus 5%).

Then verify the threshold enforcement fires:
1. Temporarily change `<minimum>{minimum_ratio}</minimum>` to `<minimum>0.99</minimum>`
2. Run `mvn test` — must exit non-zero
3. Check build output contains "Coverage checks have not been met" (JaCoCo's threshold violation message) and does NOT contain "StackOverflowError"
4. Restore `<minimum>` to the correct value from Step 1
5. Run `mvn test` again — must exit 0

The exact block to insert:

```xml
    <!-- Maven Failsafe Plugin: integration test runner for *IT.java files.
         No version: Spring Boot 3.4.4 BOM manages at 3.5.2.
         No *IT.java files exist yet — Failsafe runs zero tests; exits 0.
         argLine uses same @{argLine} late-binding as Surefire (D-06) —
         required for Phase 14 Testcontainers tests to receive JaCoCo agent.
         D-08: bound to integration-test + verify lifecycle phases. -->
    <plugin>
      <groupId>org.apache.maven.plugins</groupId>
      <artifactId>maven-failsafe-plugin</artifactId>
      <!-- No <version>: Spring Boot 3.4.4 BOM manages at 3.5.2 -->
      <executions>
        <execution>
          <goals>
            <goal>integration-test</goal>
            <goal>verify</goal>
          </goals>
        </execution>
      </executions>
      <configuration>
        <argLine>@{argLine} -XX:+EnableDynamicAgentLoading</argLine>
      </configuration>
    </plugin>
```
