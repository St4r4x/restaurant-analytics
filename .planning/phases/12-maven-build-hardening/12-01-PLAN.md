**Step 1 — Check JaCoCo 0.8.14 availability (prefer it over 0.8.13 for full Java 25 support):**

Run: `curl -s -o /dev/null -w "%{http_code}" https://repo1.maven.org/maven2/org/jacoco/jacoco-maven-plugin/0.8.14/jacoco-maven-plugin-0.8.14.pom`

If the HTTP response code is `200`, use version `0.8.14`. If not, use `0.8.13`.

**Step 2 — Add JaCoCo plugin block to pom.xml:**

Insert the following XML block into `pom.xml` inside `<build><plugins>`, immediately AFTER the closing `</plugin>` tag of the existing `maven-surefire-plugin` block and BEFORE the closing `</plugins>` tag.

Use whichever version (0.8.14 or 0.8.13) was determined in Step 1:

```xml
    <!-- JaCoCo Maven Plugin: coverage instrumentation, HTML report, and threshold check.
         Version must be explicit — Spring Boot 3.4.4 BOM does not manage jacoco-maven-plugin.
         0.8.14+ required for official Java 25 support (0.8.13 = experimental). -->
    <plugin>
      <groupId>org.jacoco</groupId>
      <artifactId>jacoco-maven-plugin</artifactId>
      <version>0.8.14</version>  <!-- replace with 0.8.13 if 0.8.14 not available on Maven Central -->
      <executions>

        <!-- Goal 1: Deposit -javaagent:jacocoagent.jar=... into the 'argLine' Maven property.
             Must run at 'initialize' phase — before test-compile — so Surefire picks it up
             via @{argLine} late-binding. Running at any later phase causes Surefire to miss it. -->
        <execution>
          <id>jacoco-prepare-agent</id>
          <phase>initialize</phase>
          <goals><goal>prepare-agent</goal></goals>
        </execution>

        <!-- Goal 2: Generate HTML/XML report at target/site/jacoco/ after tests complete.
             Excludes: pure data carriers (DTOs, JPA entities, aggregation POJOs, MongoDB domain POJOs).
             Covered: service, dao, controller, security, cache, sync, util, config (D-04, D-05). -->
        <execution>
          <id>jacoco-report</id>
          <phase>test</phase>
          <goals><goal>report</goal></goals>
          <configuration>
            <excludes>
              <!-- Exclusion format: forward-slash paths (NOT dot-notation). 'com.st4r4x.dto.*' would NOT work. -->
              <exclude>com/st4r4x/dto/**</exclude>
              <exclude>com/st4r4x/entity/**</exclude>
              <exclude>com/st4r4x/aggregation/**</exclude>
              <exclude>com/st4r4x/domain/**</exclude>
            </excludes>
          </configuration>
        </execution>

      </executions>
    </plugin>
```

**IMPORTANT:** Do NOT add the `jacoco-check` execution in this task. It is added in Plan 02, after the baseline is measured. Adding it now with a placeholder value would violate D-01 and D-02.
