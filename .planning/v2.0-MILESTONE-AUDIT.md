### Broken Flows / Defects (7 actionable)

#### DEFECT-01 — Wrong KPI URL in `landing.html` and `index.html` [CRITICAL]
- `landing.html:84` calls `fetch('/api/analytics/kpi')` — missing trailing `s`
- `index.html:142` same wrong URL
- Actual endpoint: `GET /api/analytics/kpis` (AnalyticsController line 37)
- **Effect:** KPI hero strip on landing page and auth home always shows `—`. Affects **UX-01, UX-02, STAT-01**

#### DEFECT-02 — Wrong field name `pctGradeA` vs `percentGradeA` [CRITICAL]
- `landing.html:92` reads `d.pctGradeA`; `index.html:148` same
- Backend sets key `percentGradeA` (AnalyticsController line 79)
- **Effect:** Grade A percentage tile always `—` even if URL fixed. Affects **UX-01, UX-02**

#### DEFECT-03 — Wrong response shape in `index.html` KPI handler [CRITICAL]
- `index.html:145`: `if (res.status === 'success' && res.data)` — KPI response is a flat object with no `.data` key
- `analytics.html` handles this correctly (reads `data.totalRestaurants` directly)
- **Effect:** All 4 KPI tiles on authenticated home never populate. Affects **UX-02**

#### DEFECT-04 — `loadNearby()` treats JSON object as array in `restaurant.html` [CRITICAL]
- `restaurant.html:281`: `(data || []).filter(...)` — `data` is the full `{ status, data: [...], count }` object, not an array
- Throws `TypeError`; `.catch()` hides the section silently
- Should be: `(data.data || []).filter(...)`
- **Effect:** Nearby Restaurants section is never shown. **DISC-03 is fully broken.**

#### DEFECT-05 — ADM-02 CSV download blocked by `@PreAuthorize` [CRITICAL]
- `admin.html:54`: `window.location.href='/api/inspection/at-risk/export.csv'`
- Browser navigation sends no `Authorization` header; `@PreAuthorize("hasRole('ADMIN')")` returns 403 JSON
- **Effect:** Clicking Download At-Risk CSV shows error JSON in browser tab, not a file download. **ADM-02 is broken.**
- Fix options: (a) fetchWithAuth + Blob download, or (b) make endpoint permit-all with UI-only guard

#### DEFECT-06 — `bookmarkCount` not displayed in `profile.html` [UX-04 PARTIAL]
- `/api/users/me` returns both `bookmarkCount` and `reportCount`
- `profile.html:93-100` only renders the `reportCount` stat tile
- **Effect:** UX-04 requirement ("bookmark count") is missing from the profile page

#### DEFECT-07 — SecurityConfig `permitAll` declaration is dead code [LOW]
- `SecurityConfig.java:56`: `.antMatchers("/api/inspections/**").permitAll()` (plural `inspections`)
- `InspectionController.java:30`: `@RequestMapping("/api/inspection")` (singular)
- Routes work now only by `anyRequest().permitAll()` fallthrough — silently broken if default is ever restricted

#### Minor Gap — Admin login does not redirect to `/admin`
- After login, `ROLE_ADMIN` users land on customer home (`index.html`) — not `/admin`
- `ViewController.index()` has no `ROLE_ADMIN` branch; `login.html` only redirects `ROLE_CONTROLLER` → `/dashboard`
- No formal requirement ID but implied by ADM-01 usability
